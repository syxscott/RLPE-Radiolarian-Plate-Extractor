# RLPE 使用方式对比

## CLI vs Web 界面

### 原始方式: 命令行 (CLI)

```bash
# 编写配置文件或命令行参数
python -m rlpe \
    --pdf-dir ./pdfs \
    --work-dir ./work \
    --grobid-url http://localhost:8070 \
    --ocr-backend paddleocr \
    --num-workers 4 \
    --min-panel-score 0.8 \
    --use-gemma4 \
    --llm-backend ollama \
    --export-csv results.csv \
    --export-json results.json
```

**适合**: 开发者, 有编程经验的研究者  
**难度**: ⭐⭐⭐ (中等)  
**学习曲线**: 陡峭  

### 新方式: Web 界面 (推荐)

```
1. 运行 run_web_server.py
2. 打开 http://localhost:8000
3. 拖拽上传 PDF
4. 点击几个按钮配置参数
5. 点击"开始处理"
6. 监控进度
7. 查看和导出结果
```

**适合**: 古生物学者, 非技术用户, 所有人  
**难度**: ⭐ (非常简单)  
**学习曲线**: 平坦

## 典型工作流对比

### 场景: 处理 50 篇 PDF, 手动审查, 导出结果

#### CLI 方式 (需时 ~2-3 小时)
```
0. 安装依赖                        ~ 30 分钟
1. 阅读文档和配置说明              ~ 30 分钟
2. 创建 PDF 输入文件夹              ~ 5 分钟
3. 编写或复制配置参数              ~ 10 分钟
4. 运行命令                         ~ 20 分钟 (取决于硬件)
5. 等待处理完成                    ~ 不确定 (可能 1-2 小时)
6. 手动检查 JSON 文件               ~ 30 分钟
7. 格式转换和修正 (编写脚本)       ~ 30 分钟
8. 导出 CSV                         ~ 10 分钟

总时间: 3.5-4.5 小时 ⏱️
```

#### Web 方式 (需时 ~30-40 分钟)
```
0. 启动服务器                       ~ 1 分钟
1. 打开浏览器                       ~ 1 分钟
2. 拖拽 50 个 PDF                   ~ 2 分钟
3. 点击配置按钮 (使用默认值)       ~ 2 分钟
4. 点击"开始处理"                  ~ 1 分钟
5. 监控进度                         ~ 20 分钟 (可做其他事)
6. 在表格中查看结果                 ~ 5 分钟
7. 纠正 2-3 个低置信度项            ~ 5 分钟
8. 一键导出 CSV                     ~ 1 分钟

总时间: 35-45 分钟 ⏱️
```

**时间节省**: **70-90% ⏱️** 
**难度降低**: **90%** ⬇️

## 用户体验改进

### 问题 1: 参数配置困难
```
❌ CLI 方式
$ python -m rlpe --help
# 输出 30+ 个参数选项 😵
# 不知道什么时候用哪个参数

✅ Web 方式
# 直观的表单界面
# 默认值已预设
# 参数分组清晰
# 有帮助说明文字
```

### 问题 2: 处理进度不明确
```
❌ CLI 方式
$ python process.py
# 等待...
# 什么时候完成？
# 是否卡住了？

✅ Web 方式
# 实时进度条
# 任务列表显示
# 自动刷新状态
# 完成通知提示
```

### 问题 3: 结果审查困难
```
❌ CLI 方式
# 打开 matches.jsonl
# 格式难以阅读
# 需要自己写脚本解析
# 查看图版需手动操作

✅ Web 方式
# 表格式结果显示
# 点击缩略图预览
# 搜索筛选功能
# 统计信息一目了然
```

### 问题 4: 纠正流程复杂
```
❌ CLI 方式
# 编辑 JSON 文件手动修改
# 容易出错
# 难以追踪谁做的修改

✅ Web 方式
# 表单式界面
# 字段验证
# 自动记录审核者
# 时间戳自动生成
# 修改历史清晰
```

|---|---|---|
