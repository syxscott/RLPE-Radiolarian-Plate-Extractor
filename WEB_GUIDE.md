# RLPE Web 界面使用指南

> **放射虫图版提取系统 - 网页界面**

这是一个为古生物学者设计的友好网页界面，无需命令行即可使用 RLPE 系统。

## 🚀 快速开始

### 1. 启动服务器

#### Windows 用户
```bash
# 双击运行此文件
run_web_server.bat

# 或使用 PowerShell / CMD
python run_web_server.py
```

#### Linux/macOS 用户
```bash
chmod +x run_web_server.sh
./run_web_server.sh

# 或直接使用 Python
python3 run_web_server.py
```

### 2. 打开网页
在浏览器中打开: **http://localhost:8000**

如果自动跳转失败，手动访问该地址。

### 3. 查看 API 文档
快速 API 参考: **http://localhost:8000/docs**

## 📋 功能模块说明

### 📤 上传处理 (Upload Tab)

这是主要的处理界面。

#### 步骤 1: 上传 PDF 文件
- **拖拽**: 直接拖动 PDF 文件到上传区域
- **点击**: 点击上传区域选择文件
- **支持**: 单个或批量上传多个 PDF

#### 步骤 2: 配置处理参数

**基础配置**:
- **GROBID 服务地址**: 默认 `http://localhost:8070`
  - 确保 GROBID 服务已启动
  - 用于提取 PDF 中的图版和文字
  
- **OCR 引擎**: 选择 PaddleOCR 或 EasyOCR
  - PaddleOCR: 更快，更适合中文
  - EasyOCR: 多语言支持更好

- **并发处理数**: 1-32 之间
  - 根据 CPU 核数调整 (建议 = CPU 核数)
  - 更高的并发 = 更快但更耗资源

- **Panel 置信度阈值**: 0.0-1.0
  - 更高 = 更严格的 Panel 分割
  - 推荐 0.8 (保留高质量 Panel)

**LLM 增强选项** (可选):
- 勾选 "启用 Gemma 4 后处理增强" 来使用多模态 AI 辅助
- 选择 LLM 后端:
  - **Transformers**: 需要 GPU，本地加载模型
  - **Ollama**: 使用 Ollama 管理本地模型
  - **llama.cpp**: 轻量级 C++ 推理服务器

**地质信息提取** (可选):
- 勾选 "启用地质信息 LLM 提取" 来从论文全文中提取地质数据

**调试选项**:
- 勾选 "保存中间结果" 来保存每个处理步骤的结果 (用于调试)

#### 步骤 3: 开始处理
1. 确认文件已上传
2. 检查配置参数
3. 点击 **"▶️ 开始处理"** 按钮
4. 系统将任务提交到后台处理队列

---

### 📋 任务管理 (Jobs Tab)

实时监控所有处理任务的进度。

#### 功能:
- **搜索**: 输入任务 ID 的前几个字符快速查找
- **筛选**: 按状态筛选 (队列中/处理中/已完成/失败)
- **刷新**: 手动刷新任务列表 (自动每 3 秒刷新一次)

#### 任务状态说明:
- **⏳ 队列中**: 等待处理
- **⚙️ 处理中**: 正在处理 (进度条显示进度)
- **✅ 已完成**: 处理成功
- **❌ 失败**: 处理出错

#### 任务操作:
- **📊 详情**: 查看任务详细信息
- **📈 结果**: 查看该任务的识别结果
- **❌ 取消**: 取消未完成的任务

---

### 📊 结果查看 (Results Tab)

查看和审核所有识别结果。

#### 搜索和筛选:
- **搜索框**: 输入论文 ID 或物种名称
- **下拉菜单**: 按特定任务筛选结果

#### 统计信息卡片:
- **总匹配数**: 所有识别的 Panel-物种 对
- **高置信度**: 置信度 ≥ 80% 的结果
- **已识别物种**: 不重复计算的物种数量
- **物种匹配数**: 成功识别物种的 Panel 数量

#### 结果表格:
| 列 | 说明 |
|---|---|
| 论文 ID | PDF 文件的唯一标识 |
| 图版 ID | 图版的标识号 |
| Panel 标签 | 识别的 Panel 标签 (A/B/1/2/...) |
| 物种 | 识别的拉丁学名 |
| 置信度 | AI 的置信度 (0-100%) |
| 图版缩略图 | Panel 的预览图像 |
| 操作 | 查看或纠正 |

#### 置信度颜色:
- 🟢 **绿色 (≥80%)**: 高置信度，可信
- 🟡 **黄色 (50-80%)**: 中等置信度，建议审查
- 🔴 **红色 (<50%)**: 低置信度，需要人工审查

#### 图版操作:
- **点击缩略图**: 查看完整图版 (弹出放大窗口)
- **✏️ 纠正**: 提交标注纠正

#### 导出结果:
- **💾 导出**: 将结果导出为 CSV 文件 (可在 Excel 中打开)

---

### ✏️ 标注纠正 (Correction Modal)

当识别结果不准确时，可以手动纠正并提交。

#### 纠正流程:
1. 在结果表格中点击 **✏️ 纠正** 按钮
2. 弹出纠正表单
3. 填写正确信息:
   - **物种名**: 输入正确的拉丁学名 (例: Thecopsammatium sp.)
   - **Panel 标签**: 输入正确的 Panel 标签 (例: A, B, 1, 2)
   - **审核者**: 输入您的名字 (用于数据追溯)
   - **备注**: 可选的补充说明
4. 点击 **✅ 提交纠正**
5. 纠正记录会保存到 `service_work/corrections/corrections.jsonl`

---

### ⚙️ 设置 (Settings Tab)

配置系统参数。

#### 可配置项:
- **API 基础地址**: Web 服务的地址 (默认 http://localhost:8000)
- **任务刷新间隔**: 自动刷新任务列表的时间间隔 (秒)

#### 系统信息:
显示当前系统的:
- RLPE 版本
- GROBID 服务地址
- Python 版本
- 活跃任务数

#### 保存设置:
修改后点击 **💾 保存设置** 按钮，设置会保存到浏览器本地存储。

## 📁 输出文件位置

处理完成后，结果保存在:

```
service_work/
├── {job_id}/
│   ├── pdfs/
│   │   └── your_pdf.pdf
│   ├── output/
│   │   ├── figures/          # 提取的图版
│   │   ├── panels/           # 分割的 Panel
│   │   ├── manifests/        # 匹配结果
│   │   │   └── matches.jsonl # 主要结果文件
│   │   └── tei/              # GROBID TEI XML
│   └── corrections/
│       └── corrections.jsonl  # 人工纠正记录
```

### 主要输出文件:

**matches.jsonl** - 核心输出
```json
{
  "paper_id": "pdf_hash",
  "figure_id": "fig1",
  "panel_id": "A",
  "species": "Thecopsammatium sp.",
  "confidence": 0.95,
  "bbox": [100, 200, 300, 400],
  "panel_path": "service_work/job_id/output/panels/..."
}
```

**corrections.jsonl** - 审正记录
```json
{
  "paper_id": "pdf_hash",
  "figure_id": "fig1",
  "corrected_species": "Thecopsammatium bullatum",
  "corrected_label": "A",
  "reviewer": "Dr. Smith",
  "notes": "修正了物种鉴定",
  "timestamp": "2024-01-01T12:00:00"
}
```

## 🔧 常见问题 (FAQ)

### Q: 网页打不开？
**A**: 
1. 确保服务器已启动 (看到 "Uvicorn running on..." 的消息)
2. 确保浏览器地址栏输入的是 `http://localhost:8000` (不是 https)
3. 尝试清除浏览器缓存
4. 使用其他浏览器 (Chrome/Firefox/Edge)

### Q: "无法连接" 错误？
**A**: 
1. 检查 GROBID 服务是否运行 (`http://localhost:8070/api/status`)
2. 检查防火墙设置
3. 查看服务器日志中的错误信息
4. 在"设置"标签中验证 API 基础地址

### Q: 上传很慢？
**A**:
1. 这是正常的，取决于 PDF 大小和网络
2. 大文件 (>50MB) 可能需要更长时间
3. 尝试分批上传较小的 PDF

### Q: 处理进度卡住了？
**A**:
1. 检查 GROBID 是否响应缓慢
2. 降低并发处理数
3. 查看服务器日志中是否有错误
4. 取消任务并重试

### Q: 如何使用 GPU 加速？
**A**:
1. 确保已安装 CUDA/cuDNN
2. 启用 Gemma 4 增强
3. 选择 "Transformers" LLM 后端
4. 检查 `nvidia-smi` 确认 GPU 可见

### Q: 纠正记录保存在哪里？
**A**: 
`service_work/corrections/corrections.jsonl` - 每行一条 JSON 记录

### Q: 能否导出更多格式？
**A**: 
当前支持 CSV 导出。后续可添加 Excel/JSON/XML 等格式支持。

## 🎓 工作流建议

### 初期验证 (10-20 篇 PDF)
1. 上传 10-20 篇代表性论文
2. 配置基础参数 (不启用 Gemma)
3. 观察 `output/panels/` 中的分割效果
4. 在结果表格中审查 2-3 个样本
5. 检查置信度分布

### 小规模标注 (30-50 篇 PDF)
1. 增加到 30-50 篇 PDF
2. 逐一审查低置信度结果 (<70%)
3. 使用 **✏️ 纠正** 功能修正错误
4. 统计错误率和 Panel 检测成功率

### 启用 LLM 增强 (全量批处理)
1. 配置 Gemma 4 后端
2. 启用地质信息提取 (可选)
3. 全量处理所有论文
4. 导出结果进行最终审核
